# Flashcards App - План разработки

## Цель
PWA-приложение для изучения карточек по системе интервального повторения (Anki-подобное).

## Технологии
- HTML5 + Tailwind CSS (CDN)
- Vanilla JavaScript
- localStorage для хранения прогресса
- Service Worker для офлайн-режима

## Структура проекта
```
flashcards-app/
├── index.html          # Главная страница с карточками
├── stats.html          # Страница статистики
├── app.js              # Основная логика
├── data.js             # Данные карточек (из YAML)
├── sw.js               # Service Worker
├── manifest.json       # PWA манифест
└── PLAN.md             # Этот файл
```

## Алгоритм повторения (упрощённый SM-2)

Каждая карточка имеет:
- `interval` — дней до следующего показа (1, 3, 7, 14, 30...)
- `easeFactor` — множитель сложности (2.5 по умолчанию)
- `nextReview` — timestamp следующего повторения

### Логика кнопок:
- **Снова (Again)**: interval = 1 мин, easeFactor -= 0.2
- **Помню (Good)**: interval *= easeFactor
- **Легко (Easy)**: interval *= easeFactor * 1.3, easeFactor += 0.1

## Открытые вопросы

1. **Цветовая схема?**
   - Светлая / Тёмная / Автоматически по системе?

2. **Порядок карточек?**
   - Случайный или по категориям?
   - Возможность фильтровать по категориям?

3. **Дневной лимит?**
   - Ограничивать количество новых карточек в день? (как в Anki)
   - Или показывать все подряд?

4. **Свайпы?**
   - Добавить жесты свайпа (влево = снова, вправо = помню)?

5. **Звуки/вибрация?**
   - Нужна ли тактильная обратная связь?

6. **Название приложения?**
   - "Карточки ВКР"? "Flashcards"? Другое?
